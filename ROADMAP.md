# ROADMAP

## Phase 0: Foundation

- establish brand system and dark-first product shell
- initialize web, API, and docs repositories
- define environment contracts, CI conventions, and architecture boundaries
- document prompt, content, and platform strategy

## Phase 1: MVP Discovery Loop

- launch onboarding to capture city and taste preferences
- ship recommendation feed backed by seeded data
- support place cards, event cards, save actions, and map view
- integrate AI summaries and tagging for initial content enrichment

## Phase 2: Freshness and Personalization

- add event ingestion from curated and partner sources
- introduce preference memory and behavioral ranking signals
- improve search with semantic intent mapping
- add editorial collections for key cities

## Phase 3: Expansion Layer

- mobile application with shared contracts from the existing API
- creator or partner submission workflows
- notifications for city moments and timely events
- B2B or affiliate surfaces where strategically aligned

## Operating Principles

- bias toward fast iteration with durable architecture
- keep recommendation quality ahead of breadth
- instrument product learning from the start
- treat content freshness as a product feature, not a support task
